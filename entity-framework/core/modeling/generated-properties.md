### <a name="no-value-generation"></a><span data-ttu-id="c9ba0-108">Aucune génération de valeur</span><span class="sxs-lookup"><span data-stu-id="c9ba0-108">No value generation</span></span>

> <span data-ttu-id="c9ba0-121">Par exemple, lors de l’utilisation de SQL Server, les valeurs sont générées automatiquement pour les propriétés de `GUID` (à l’aide de l’algorithme de GUID séquentiel SQL Server).</span><span class="sxs-lookup"><span data-stu-id="c9ba0-121">For example, when using SQL Server, values will be automatically generated for `GUID` properties (using the SQL Server sequential GUID algorithm).</span></span> <span data-ttu-id="c9ba0-122">Toutefois, si vous spécifiez qu’une propriété `DateTime` est générée lors de l’ajout, vous devez configurer un moyen de générer les valeurs.</span><span class="sxs-lookup"><span data-stu-id="c9ba0-122">However, if you specify that a `DateTime` property is generated on add, then you must setup a way for the values to be generated.</span></span> <span data-ttu-id="c9ba0-123">Pour ce faire, vous pouvez configurer une valeur par défaut de `GETDATE()`, voir [valeurs par défaut](relational/default-values.md).</span><span class="sxs-lookup"><span data-stu-id="c9ba0-123">One way to do this, is to configure a default value of `GETDATE()`, see [Default Values](relational/default-values.md).</span></span>
