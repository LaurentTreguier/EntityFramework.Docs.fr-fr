<span data-ttu-id="9d297-106">Le .NET Framework prend en charge la génération d’images natives pour les applications et les bibliothèques managées afin d’aider les applications à démarrer plus rapidement et, dans certains cas, à utiliser moins de mémoire.</span><span class="sxs-lookup"><span data-stu-id="9d297-106">The .NET Framework supports the generation of native images for managed applications and libraries as a way to help applications start faster and also in some cases use less memory.</span></span> <span data-ttu-id="9d297-107">Les images natives sont créées en traduisant des assemblys de code managé en fichiers contenant des instructions machine natives avant l’exécution de l’application, ce qui allège le compilateur .NET JIT (juste-à-temps) d’avoir à générer les instructions natives au moment de l’exécution de l’application.</span><span class="sxs-lookup"><span data-stu-id="9d297-107">Native images are created by translating managed code assemblies into files containing native machine instructions before the application is executed, relieving the .NET JIT (Just-In-Time) compiler from having to generate the native instructions at application runtime.</span></span>

## <a name="how-to-use-ngenexe"></a><span data-ttu-id="9d297-112">Comment utiliser NGen.exe</span><span class="sxs-lookup"><span data-stu-id="9d297-112">How to use NGen.exe</span></span>  

<span data-ttu-id="9d297-113">La fonction la plus simple de l’outil NGen.exe consiste à « installer » (autrement dit, à créer et à conserver sur le disque) des images natives pour un assembly et toutes ses dépendances directes.</span><span class="sxs-lookup"><span data-stu-id="9d297-113">The most basic function of the NGen.exe tool is to “install” (that is, to create and persist to disk) native images for an assembly and all its direct dependencies.</span></span> <span data-ttu-id="9d297-114">Voici comment y parvenir :</span><span class="sxs-lookup"><span data-stu-id="9d297-114">Here is how you can achieve that:</span></span>

   ``` console
   cd <*Assemblies location*>  
   ```

3. <span data-ttu-id="9d297-117">En fonction de votre système d’exploitation et de la configuration de l’application, vous devrez peut-être générer des images natives pour l’architecture 32 bits, l’architecture 64 bits ou pour les deux.</span><span class="sxs-lookup"><span data-stu-id="9d297-117">Depending on your operating system and the application’s configuration you might need to generate native images for 32 bit architecture, 64 bit architecture or for both.</span></span>


   <span data-ttu-id="9d297-119">Pour une exécution de 64 bits :</span><span class="sxs-lookup"><span data-stu-id="9d297-119">For 64 bit run:</span></span>
  
   ``` console
   %WINDIR%\Microsoft.NET\Framework64\v4.0.30319\ngen install <Assembly name>  
   ```

> [!TIP]
> <span data-ttu-id="9d297-120">La génération d’images natives pour une architecture incorrecte est une erreur très courante.</span><span class="sxs-lookup"><span data-stu-id="9d297-120">Generating native images for the wrong architecture is a very common mistake.</span></span> <span data-ttu-id="9d297-121">En cas de doute, vous pouvez simplement générer des images natives pour toutes les architectures qui s’appliquent au système d’exploitation installé sur l’ordinateur.</span><span class="sxs-lookup"><span data-stu-id="9d297-121">When in doubt you can simply generate native images for all the architectures that apply to the operating system installed in the machine.</span></span>

## <a name="when-to-use-ngenexe"></a><span data-ttu-id="9d297-123">Quand utiliser NGen.exe</span><span class="sxs-lookup"><span data-stu-id="9d297-123">When to use NGen.exe</span></span>  

<span data-ttu-id="9d297-124">Lorsqu’il s’agit de déterminer les assemblys pour lesquels générer des images natives dans une application basée sur EF version 6 ou ultérieure, vous devez prendre en compte les options suivantes :</span><span class="sxs-lookup"><span data-stu-id="9d297-124">When it comes to decide which assemblies to generate native images for in an application based on EF version 6 or greater, you should consider the following options:</span></span>  

- <span data-ttu-id="9d297-125">**L’assembly principal du runtime EF, EntityFramework.dll**: une application basée sur EF classique exécute une quantité significative de code à partir de cet assembly au démarrage ou lors de son premier accès à la base de données.</span><span class="sxs-lookup"><span data-stu-id="9d297-125">**The main EF runtime assembly, EntityFramework.dll**: A typical EF based application executes a significant amount of code from this assembly on startup or on its first access to the database.</span></span> <span data-ttu-id="9d297-126">Par conséquent, la création d’images natives de cet assembly produit les plus grandes améliorations des performances de démarrage.</span><span class="sxs-lookup"><span data-stu-id="9d297-126">Consequently, creating native images of this assembly will produce the biggest gains in startup performance.</span></span>  
- <span data-ttu-id="9d297-127">**Tout assembly de fournisseur EF utilisé par votre application : le**temps de démarrage peut également tirer parti de la génération d’images natives.</span><span class="sxs-lookup"><span data-stu-id="9d297-127">**Any EF provider assembly used by your application**: Startup time can also benefit slightly from generating native images of these.</span></span> <span data-ttu-id="9d297-128">Par exemple, si l’application utilise le fournisseur EF pour SQL Server vous souhaitez générer une image native pour EntityFramework.SqlServer.dll.</span><span class="sxs-lookup"><span data-stu-id="9d297-128">For example, if the application uses the EF provider for SQL Server you will want to generate a native image for EntityFramework.SqlServer.dll.</span></span>
