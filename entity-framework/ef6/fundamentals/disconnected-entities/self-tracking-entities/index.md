<span data-ttu-id="b20a7-123">Tenez compte des éléments suivants lors de l'utilisation d'entités de suivi automatique :</span><span class="sxs-lookup"><span data-stu-id="b20a7-123">Consider the following when working with self-tracking entities:</span></span>

    > <span data-ttu-id="b20a7-131">Si des objets dans votre graphique contiennent des propriétés avec des valeurs générées par la base de données (par exemple, des valeurs d’identité ou de concurrence), l’Entity Framework remplace les valeurs de ces propriétés par les valeurs générées par la base de données après l’appel de la méthode **SaveChanges**.</span><span class="sxs-lookup"><span data-stu-id="b20a7-131">If objects in your graph contain properties with database-generated values (for example, identity or concurrency values), Entity Framework will replace values of these properties with the database-generated values after the **SaveChanges** method is called.</span></span> <span data-ttu-id="b20a7-132">Vous pouvez implémenter votre opération de service pour retourner des objets enregistrés ou une liste de valeurs de propriété générées pour les objets au client.</span><span class="sxs-lookup"><span data-stu-id="b20a7-132">You can implement your service operation to return saved objects or a list of generated property values for the objects back to the client.</span></span> <span data-ttu-id="b20a7-133">Le client devra ensuite remplacer les instances d'objet ou valeurs de propriété d'objet par les objets ou valeurs de propriété retournés à partir de l'opération de service.</span><span class="sxs-lookup"><span data-stu-id="b20a7-133">The client would then need to replace the object instances or object property values with the objects or property values returned from the service operation.</span></span>

    > <span data-ttu-id="b20a7-139">Le fait que la propriété de navigation de référence ne soit pas synchronisée avec l'objet principal approprié peut poser un problème si vous avez spécifié la suppression en cascade dans la relation de clé étrangère.</span><span class="sxs-lookup"><span data-stu-id="b20a7-139">Not having a reference navigation property synchronized with the appropriate principal object could be an issue if you have specified cascade delete on the foreign key relationship.</span></span> <span data-ttu-id="b20a7-140">Si vous supprimez l'objet principal, la suppression ne sera pas propagée aux objets dépendants.</span><span class="sxs-lookup"><span data-stu-id="b20a7-140">If you delete the principal, the delete will not be propagated to the dependent objects.</span></span> <span data-ttu-id="b20a7-141">Si vous avez spécifié des suppressions en cascade, utilisez les propriétés de navigation pour modifier les relations au lieu de définir la propriété de clé étrangère.</span><span class="sxs-lookup"><span data-stu-id="b20a7-141">If you have cascade deletes specified, use navigation properties to change relationships instead of setting the foreign key property.</span></span>

## <a name="security-considerations"></a><span data-ttu-id="b20a7-146">Considérations relatives à la sécurité</span><span class="sxs-lookup"><span data-stu-id="b20a7-146">Security Considerations</span></span>  

<span data-ttu-id="b20a7-147">Les considérations de sécurité suivantes doivent être prises en compte quand vous utilisez les entités de suivi automatique :</span><span class="sxs-lookup"><span data-stu-id="b20a7-147">The following security considerations should be taken into account when working with self-tracking entities:</span></span>  

- <span data-ttu-id="b20a7-148">Un service ne doit pas compter sur les requêtes pour récupérer ou mettre à jour les données d'un client non approuvé ou via un canal non approuvé.</span><span class="sxs-lookup"><span data-stu-id="b20a7-148">A service should not trust requests to retrieve or update data from a non-trusted client or through a non-trusted channel.</span></span> <span data-ttu-id="b20a7-149">Un client doit être authentifié : un canal sécurisé doit être utilisé ou bien une enveloppe de message.</span><span class="sxs-lookup"><span data-stu-id="b20a7-149">A client must be authenticated: a secure channel or message envelope should be used.</span></span> <span data-ttu-id="b20a7-150">Les requêtes de clients pour mettre à jour ou récupérer des données doivent être validées pour vérifier qu'elles sont conformes aux modifications attendues et légitimes pour le scénario donné.</span><span class="sxs-lookup"><span data-stu-id="b20a7-150">Clients' requests to update or retrieve data must be validated to ensure they conform to expected and legitimate changes for the given scenario.</span></span>
